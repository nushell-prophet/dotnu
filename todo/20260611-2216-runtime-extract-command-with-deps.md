# Runtime-извлечение команды с каскадом зависимостей в один файл

А так напиши todo на создание команды, которая загружает выбранную команду из модуля в память и каскадно выгружает все связанные команды, чтобы создать один файл в котором все embeded.

Возможно такое?

## Task description (extended version)

Да, возможно. Runtime-аналог `extract-command-code`: вместо парсинга файла —
импорт модуля в чистом `nu -n` и выгрузка тел команд через `view source`.

Зачем, если `extract-command-code` уже есть: статический парсинг работает с
одним файлом-скриптом. Runtime-интроспекция видит модуль так, как его видит
Nushell после резолва — `export use`, подмодули, переименованные экспорты.
Это путь к самодостаточному (self-contained) файлу из любого модуля.

Механика (проверена в сессии 2026-06-11):

1. **Safety-скан перед импортом.** `use` выполняет `export-env` блоки —
   свои и у транзитивно подключаемых модулей. Это единственный канал
   выполнения кода при импорте: произвольный код на верхнем уровне модуля —
   parse error, `const` ограничен const-safe командами. Поэтому до импорта —
   статическая проверка (AST, не regex) всех `.nu` файлов модуля на
   `export-env`. Нашли — ошибка со списком файлов; флаг
   `--allow-export-env` разрешает продолжить.
2. **Копия с export-ификацией.** Приватные `def` (без `export`)
   недоступны снаружи: их нет в `scope commands`, `view source` на них
   падает (проверено). `view source <имя-модуля>` возвращает сырой текст
   файла — преимуществ перед `open` нет. Обход: скопировать директорию
   модуля во временную, во всех `.nu` файлах переписать `def` ->
   `export def` (по AST; тела не меняются, относительные `use` внутри
   копии сохраняются).
3. **Выгрузка в один заход.** Один вызов
   `nu -n -c 'use <копия> *; ...'` (именно `*` — без него команды
   намespace'ятся как `<mod> <cmd>` и `view source <cmd>` не находит их):
   взять список команд из `scope commands`, для каждой — `view source`,
   вернуть record `name -> source` (nuon). Один процесс вместо вызова
   на команду.
4. **Каскад.** От выбранной команды — BFS по record'у: в тексте каждой
   команды найти вызовы (AST-токены, пересечение с именами из record'а),
   добавить достижимые. Это пересекается с логикой `dependencies`.
5. **Сборка файла.** Достижимое множество в топологическом порядке —
   зависимости раньше зависимых (парсер Nushell требует def до вызова).
   Команды, бывшие приватными в исходнике, в выходном файле снова `def`
   без `export` (export-ификация — только трюк для извлечения).

## Requirements

- [ ] Сигнатура вида: `<command> <module_path> <command_name> [--allow-export-env]`
- [ ] Без флага и с `export-env` в любом файле модуля — ошибка со списком
      файлов, импорт не происходит
- [ ] Скан покрывает файлы модуля; транзитивные `use` локальных файлов вне
      директории модуля — хотя бы задокументированное ограничение
- [ ] Выходной файл самодостаточен: `nu -n` парсит его без ошибок, целевая
      команда вызывается
- [ ] Приватные зависимости (`def` без `export`) тоже извлекаются и
      попадают в выходной файл — как `def`, без `export`
- [ ] Порядок определений топологический; дубликатов нет
- [ ] Тесты на: простой каскад, модуль с подмодулем (`export use`),
      отказ на `export-env`, проход с `--allow-export-env`

## Implementation plan

- [ ] Шаг 1: safety-скан — найти `export-env` через AST
      (`split-statements` или `nu --ide-ast`) по `.nu` файлам модуля
- [ ] Шаг 2: временная копия модуля с переписыванием `def` ->
      `export def` по AST во всех `.nu` файлах
- [ ] Шаг 3: выгрузка `name -> source` одним `nu -n -c` вызовом
      (`use <копия> *` + scope commands + view source, вывод nuon)
- [ ] Шаг 4: каскад — выделение вызовов из текста команды (AST-токены),
      BFS по record'у
- [ ] Шаг 5: топосортировка, сборка текста (приватные снова без
      `export`), `--output` / stdout
- [ ] Шаг 6: тесты в `tests/test_commands.nu`: фикстура-модуль с
      подмодулем и приватным helper'ом, фикстура с `export-env`
- [ ] Шаг 7: решить про экспорт в `mod.nu` (публичная команда или
      внутренняя на первом этапе)
